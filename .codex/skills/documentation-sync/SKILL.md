# documentation-sync

## Purpose
Při každé změně projektu vždy aktualizuj dokumentaci tak, aby odpovídala aktuálnímu stavu aplikace, funkcí a chování.

## Instructions
- Každou změnu v kódu nebo chování promítni také do dokumentace.
- Pokud se změní UI, funkce, workflow nebo pravidla, uprav i příslušný popis v dokumentech.
- Dokumentace nesmí zůstat zastaralá po provedené změně.
- Před dokončením úkolu zkontroluj, zda dokumentace odpovídá výslednému stavu.
